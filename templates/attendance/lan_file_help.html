üõ°Ô∏è Security Features

üîê Network-Level Security

The system automatically validates that all requests come from approved LAN IP ranges:

  • 10.0.0.0/8 - Class A private networks
  • 172.16.0.0/12 - Class B private networks
  • 192.168.0.0/16 - Class C private networks
  • 127.0.0.0/8 - Loopback addresses

Key Security Features:

  • IP Validation: Only LAN IP addresses can access the system
  • Employee Authentication: All activities tied to employee IDs
  • Department Isolation: Files shared based on organizational structure
  • Access Logging: Complete audit trail of all file access
  • File Integrity: MD5 hash verification prevents corruption
  • Confidentiality Marking: Special handling for sensitive files

üì§ Uploading Files

Step-by-Step Upload Process:

  1. Select Files: Click the upload area and choose files (multiple selection supported)
  2. Employee Information: Enter your Employee ID and full name
  3. Department: Select your department from the dropdown
  4. Access Level: Choose who can access your files
  5. Description: Add an optional description
  6. Tags: Add comma-separated tags for easier searching
  7. Confidential: Mark sensitive files as confidential
  8. Upload: Click "Upload to LAN" to start the process

‚ö†Ô∏è Upload Limits

  • Maximum file size: 500MB per file
  • Multiple files can be uploaded simultaneously
  • Files are automatically scanned for duplicates
  • Progress is shown for each file during upload

üìÅ Browsing & Downloading Files

Accessing Files:

  1. Navigate to the "Browse Files" tab
  2. Enter your Employee ID and Department
  3. Click "Load Files" to see available files
  4. Click "Download" to download files you have access to

File Information Displayed:

  • Filename: Original file name
  • File Size: Size in appropriate units (KB, MB, GB)
  • Uploader: Name of the employee who uploaded the file
  • Upload Date: When the file was uploaded
  • File Type: Category of the file
  • Download Count: How many times the file has been downloaded
  • Confidential Status: Whether the file is marked as confidential
  • Description: Optional description provided by uploader
  • Tags: Keywords for easier categorization

‚úÖ Best Practices

For File Uploaders:

  • Use descriptive filenames that clearly indicate content
  • Add meaningful descriptions to help colleagues understand the file purpose
  • Use relevant tags for better searchability
  • Choose appropriate access levels based on file sensitivity
  • Mark confidential files appropriately
  • Regularly review and clean up old files

For File Downloaders:

  • Only download files you need for work purposes
  • Respect confidentiality markings
  • Do not share downloaded files outside the approved network
  • Report any inappropriate content to system administrators

Security Best Practices:

  • Never attempt to access the system from outside the company network
  • Always use your actual employee ID and name
  • Do not share login credentials with others
  • Be cautious with files marked as confidential
  • Report security concerns immediately

üîß Troubleshooting

Common Issues and Solutions:

‚ùå "Access denied: Must be connected to company LAN"

Solution: Ensure you are connected to the company network. VPN connections may not work depending on configuration.

‚ùå "File type not allowed"

Solution: Check the supported file types list above. Contact IT if you need support for additional file types.

‚ùå "File already exists (duplicate detected)"

Solution: The system detected an identical file already exists. Check if someone else has uploaded the same file.

‚ùå Upload fails or times out

Solution: Large files may take time to upload. Ensure stable network connection and try again. Contact IT for files larger than 500MB.

‚ùå No files visible in browse tab

Solution: Verify your Employee ID and Department are correct. You may not have access to any files based on your current permissions.

Getting Help:

If you continue to experience issues:

  1. Check that you're connected to the company LAN
  2. Verify your employee information is correct
  3. Try refreshing the page
  4. Contact your IT department with specific error messages
